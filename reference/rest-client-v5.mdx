`RestClientV5` is the single unified REST client for every Bybit V5 endpoint. It covers public market data, authenticated trading, position management, account configuration, asset transfers, user administration, and specialised categories such as Earn, Broker, Institutional Lending, and P2P. All methods return typed `Promise` values, and authentication is handled automatically whenever you supply `key` and `secret` during construction.

## Installation & Import

```bash
npm install bybit-api
```

```typescript
import { RestClientV5 } from 'bybit-api';
```

<Note>
  **Looking for the legacy Spot V3 client?** The SDK also exports `SpotClientV3` for backwards compatibility with Bybit's older V3 Spot REST endpoints. Bybit has deprecated those endpoints in favour of the unified V5 API, so all new integrations should use `RestClientV5` with `category: 'spot'` instead.

  ```typescript
  import { SpotClientV3 } from 'bybit-api'; // legacy — prefer RestClientV5
  ```
</Note>

## Constructor

```typescript
const client = new RestClientV5(options?: RestClientOptions);
```

<ParamField body="options" type="RestClientOptions" optional>
  Configuration object for the REST client. All fields are optional.

  <Expandable title="RestClientOptions fields">
    <ParamField body="key" type="string" optional>Your API key.</ParamField>
    <ParamField body="secret" type="string" optional>Your API secret.</ParamField>
    <ParamField body="testnet" type="boolean" optional>Connect to Bybit testnet. Default `false`.</ParamField>
    <ParamField body="demoTrading" type="boolean" optional>Use Bybit V5 demo trading environment. Default `false`.</ParamField>
    <ParamField body="apiRegion" type="APIRegion" optional>Override the API region (`'default' | 'bytick' | 'NL' | 'TK' | 'KZ' | 'HK' | 'GE' | 'UAE' | 'EU'`).</ParamField>
    <ParamField body="recv_window" type="number" optional>Override the maximum request window in milliseconds.</ParamField>
    <ParamField body="baseUrl" type="string" optional>Override the API base URL, e.g. `'https://api.bytick.com'`.</ParamField>
    <ParamField body="enable_time_sync" type="boolean" optional>Automatically sync clock drift with Bybit servers. Default `false`.</ParamField>
    <ParamField body="parseAPIRateLimits" type="boolean" optional>Parse per-endpoint rate limit headers from responses. Default `false`.</ParamField>
    <ParamField body="throwExceptions" type="boolean" optional>Throw on failed REST API responses instead of returning them. Default `false`.</ParamField>
    <ParamField body="customSignMessageFn" type="(message: string, secret: string) => Promise<string>" optional>Provide a custom HMAC signing function (e.g. Node's `crypto.createHmac`).</ParamField>
  </Expandable>
</ParamField>

### Instantiation examples

<CodeGroup>
```typescript Public-only (no auth)
import { RestClientV5 } from 'bybit-api';

const client = new RestClientV5();
```

```typescript Authenticated – mainnet
import { RestClientV5 } from 'bybit-api';

const client = new RestClientV5({
  key: 'YOUR_API_KEY',
  secret: 'YOUR_API_SECRET',
});
```

```typescript Authenticated – testnet
import { RestClientV5 } from 'bybit-api';

const client = new RestClientV5({
  key: 'YOUR_API_KEY',
  secret: 'YOUR_API_SECRET',
  testnet: true,
});
```

```typescript Demo trading
import { RestClientV5 } from 'bybit-api';

const client = new RestClientV5({
  key: 'YOUR_API_KEY',
  secret: 'YOUR_API_SECRET',
  demoTrading: true,

```
</CodeGroup>

<Note>
  The `key` and `secret` fields are only required when calling authenticated (🔒) endpoints. Public endpoints work without them.

### Example — place and cancel an order

```typescript
// Submit a limit buy order
const order = await client.submitOrder({
  category: 'linear',
  symbol: 'BTCUSDT',
  side: 'Buy',
  orderType: 'Limit',
  qty: '0.01',
  price: '30000',
  timeInForce: 'GTC',
});

// Cancel the same order
await client.cancelOrder({
  category: 'linear',
  symbol: 'BTCUSDT',
  orderId: order.result.orderId,
});
```

### Example — batch orders

```typescript
const result = await client.batchSubmitOrders('linear', [
  { symbol: 'BTCUSDT', side: 'Buy', orderType: 'Limit', qty: '0.01', price: '29000', timeInForce: 'GTC' },
  { symbol: 'ETHUSDT', side: 'Sell', orderType: 'Limit', qty: '0.1', price: '2000', timeInForce: 'GTC' },
]);
```
